# RBT: Платформа оркестрации туннелей

[![License: MIT](https://img.shields.io/badge/License-MIT-yellow.svg)](https://opensource.org/licenses/MIT)
[![Rust](https://img.shields.io/badge/Rust-1.75+-orange.svg)](https://www.rust-lang.org/)
[![Version](https://img.shields.io/badge/Version-0.0.2-blue.svg)](https://github.com/EHSANKiNG/RBT)

RBT — это платформа для туннелирования «сервер-сервер», ориентированная на инженерные исследования. Предназначена для законного администрирования инфраструктуры, контролируемых экспериментов и технической оценки.

RBT отлично работает в сетях с высокими потерями благодаря использованию современных протоколов (QUIC) и оптимизаций уровня ядра (Zero-Copy TCP).

## 🛠️ Технологический стек
- **Ядро**: Rust (Высокая производительность, безопасность памяти, асинхронный супервизор)
- **Фронтенд**: React, Tailwind CSS, Lucide Icons
- **Бэкенд**: Node.js/Express, WebSocket для телеметрии в реальном времени
- **Туннелирование**: QUIC (UDP датаграммы), Linux `splice()` (Zero-Copy TCP)

## 📥 Установка
Вы можете установить RBT и все его зависимости с помощью нашего скрипта автоматической установки. Этот скрипт определяет вашу ОС, устанавливает необходимые системные зависимости (например, `rustup`, `node`, `npm`) и настраивает среду RBT.

```bash
curl -fsSL https://raw.githubusercontent.com/EHSANKiNG/RBT/main/install.sh | bash
```

*Примечание: Этот скрипт требует прав root для установки системных зависимостей.*

## ⚙️ Конфигурация и переменные окружения
После установки настройте следующие переменные окружения в файле `.env`:

| Переменная | Описание |
| :--- | :--- |
| `HCAPTCHA_SITE_KEY` | Ключ сайта от [hCaptcha](https://www.hcaptcha.com/). |
| `HCAPTCHA_SECRET` | Секретный ключ от [hCaptcha](https://www.hcaptcha.com/). |
| `JWT_SECRET` | Сильная случайная строка для подписи токенов аутентификации. **Сгенерировать здесь:** [jwtsecrets.com](https://jwtsecrets.com/#generator). |
| `SMTP_HOST` | Имя хоста SMTP-сервера для уведомлений. |
| `SMTP_PORT` | Порт SMTP-сервера (например, 587). |
| `SMTP_USER` | Имя пользователя SMTP. |
| `SMTP_PASS` | Пароль SMTP. |
| `NOTIFICATION_EMAIL` | Адрес электронной почты для получения оповещений. |

## 📖 Механика туннелирования
- **Анти-DPI обфускация**: Маскировка заголовков пакетов для защиты от систем DPI.
- **Динамическая смена портов**: Обход блокировок портов с использованием синхронизации на основе TOTP.
- **QUIC UDP датаграммы**: Устранение блокировки начала очереди для UDP-трафика.
- **Zero-Copy TCP**: Интеграция `splice()` уровня ядра Linux для максимальной пропускной способности.

## 💻 Справочник CLI
Интерфейс командной строки `rbt` — основной инструмент для управления вашими туннелями.

| Команда | Описание |
| :--- | :--- |
| `rbt link-add --name <name> --listen <addr> --target <addr> --proto <tcp\|udp>` | Создать новый туннель. |
| `rbt apply` | Применить конфигурацию и запустить супервизор. |
| `rbt logs -f` | Потоковый просмотр логов. |
| `rbt stats --json` | Получить статистику трафика в формате JSON. |
