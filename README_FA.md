# RBT: پلتفرم ارکستراسیون تونل

[![License: MIT](https://img.shields.io/badge/License-MIT-yellow.svg)](https://opensource.org/licenses/MIT)
[![Rust](https://img.shields.io/badge/Rust-1.75+-orange.svg)](https://www.rust-lang.org/)
[![Version](https://img.shields.io/badge/Version-0.0.2-blue.svg)](https://github.com/EHSANKiNG/RBT)

RBT یک پلتفرم تونل‌زنی سرور به سرور است که با تمرکز بر مهندسی و تحقیق توسعه یافته است. این ابزار برای مدیریت قانونی زیرساخت‌ها، آزمایش‌های کنترل‌شده و ارزیابی فنی طراحی شده است.

RBT با بهره‌گیری از پروتکل‌های مدرن (QUIC) و بهینه‌سازی‌های سطح کرنل (Zero-Copy TCP)، در شبکه‌های با اتلاف بالا عملکرد عالی دارد.

## 🛠️ پشته تکنولوژی (Tech Stack)
- **هسته**: Rust (عملکرد بالا، ایمن از نظر حافظه، ناظر ناهمگام)
- **فرانت‌اند**: React, Tailwind CSS, Lucide Icons
- **بک‌اند**: Node.js/Express, WebSocket برای تله‌متری زنده
- **تونل‌زنی**: QUIC (UDP Datagrams), Linux `splice()` (Zero-Copy TCP)

## 📥 نصب
شما می‌توانید RBT و تمامی نیازمندی‌های آن را با استفاده از اسکریپت نصب خودکار ما نصب کنید. این اسکریپت سیستم‌عامل شما را تشخیص داده، نیازمندی‌های سیستمی (مانند `rustup`, `node`, `npm`) را نصب کرده و محیط RBT را آماده می‌کند.

```bash
curl -fsSL https://raw.githubusercontent.com/EHSANKiNG/RBT/main/install.sh | bash
```

*توجه: این اسکریپت برای نصب نیازمندی‌های سیستمی به دسترسی روت (root) نیاز دارد.*

## ⚙️ پیکربندی و متغیرهای محیطی
پس از نصب، متغیرهای محیطی زیر را در فایل `.env` خود تنظیم کنید:

| متغیر | توضیحات |
| :--- | :--- |
| `HCAPTCHA_SITE_KEY` | کلید سایت از [hCaptcha](https://www.hcaptcha.com/). |
| `HCAPTCHA_SECRET` | کلید مخفی از [hCaptcha](https://www.hcaptcha.com/). |
| `JWT_SECRET` | یک رشته قوی و تصادفی برای امضای توکن‌های احراز هویت. **تولید از اینجا:** [jwtsecrets.com](https://jwtsecrets.com/#generator). |
| `SMTP_HOST` | نام میزبان سرور SMTP برای اعلان‌ها. |
| `SMTP_PORT` | پورت سرور SMTP (مثلاً 587). |
| `SMTP_USER` | نام کاربری SMTP. |
| `SMTP_PASS` | رمز عبور SMTP. |
| `NOTIFICATION_EMAIL` | آدرس ایمیل دریافت‌کننده هشدارها. |

## 🚀 رابط کاربری تعاملی (CLI)

```bash

```
*(نیاز به نصب Node.js دارد)*

## 📖 مکانیسم تونل‌زنی
- **مخفی‌سازی ترافیک (Anti-DPI):** ماسک کردن هدرهای بسته برای جلوگیری از شناسایی توسط سیستم‌های DPI.
- **تغییر پورت پویا (Port Hopping):** دور زدن مسدودسازی پورت‌ها با استفاده از همگام‌سازی TOTP.
- **پشتیبانی از QUIC:** حذف مسدودسازی برای ترافیک UDP در شبکه‌های با اتلاف بالا.
- **TCP Zero-Copy:** استفاده از `splice()` کرنل لینوکس برای حداکثر کارایی و حداقل تأخیر.

## 💻 راهنمای CLI
رابط خط فرمان `rbt` ابزار اصلی برای مدیریت تونل‌های شماست.

| دستور | توضیحات |
| :--- | :--- |
| `rbt link-add --name <name> --listen <addr> --target <addr> --proto <tcp\|udp>` | ایجاد یک تونل جدید. |
| `rbt apply` | اعمال تنظیمات و شروع ناظر (Supervisor). |
| `rbt logs -f` | مشاهده زنده لاگ‌ها. |
| `rbt stats --json` | دریافت آمار ترافیک به صورت JSON. |
